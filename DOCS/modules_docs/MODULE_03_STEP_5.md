MODULE_03_STEP_5 — DIRECTORY CARD VIEW
1. Назначение

Этот шаг отвечает за:

OBJECT LIST → открытие полной карточки

Это точка, где пользователь:

понимает, кто перед ним

решает: взаимодействовать или выйти

2. Место в маршруте
MAIN MENU
→ Услуги
→ CATEGORY MENU
→ OBJECT LIST
→ CARD VIEW
3. Источник данных

Только через loader:

get_object_by_id(card_id)

❗ UI не читает JSON напрямую

4. Главное правило карточки
КАРТОЧКА = ТОЧКА ДЕЙСТВИЯ

Не витрина. Не каталог.
А место, где пользователь принимает решение.

5. Формат экрана

Карточка должна быть:

читаемой

короткой

без мусора

6. Структура текста карточки

Базовый формат:

<Название>

<Краткое описание>

📍 Район: <district>
📌 Локация: <location>

🛠 Услуги:
- услуга 1
- услуга 2
- услуга 3

📞 Контакты:
телефон / WhatsApp / Telegram / Instagram
7. Что показывать обязательно
name
title / short_description
location / district
services (списком)
контакты (если есть)
8. Что НЕ показывать

❌ лишний маркетинг
❌ длинные тексты
❌ “лучший мастер города”
❌ цены (на этом этапе)
❌ отзывы
❌ рейтинг

9. Фото

На baseline:

1 изображение

Telegram:

send_photo + caption

Если нет фото:

просто текст

10. Кнопки карточки

Минимальный набор:

📨 Оставить запрос
📞 Связаться
⬅️ Назад
11. Логика кнопок
📩 Оставить запрос

Работает если:

request_enabled = true

И есть:

partner_chat_id

Если нет:

Партнёр пока не подключил приём заявок.
📞 Связаться

Открывает:

телефон

WhatsApp

Telegram

Instagram

(в зависимости от того, что заполнено)

⬅️ Назад

Возвращает в:

OBJECT LIST той же категории
12. Edge case — карточка не найдена

Если:

get_object_by_id = None

Показываем:

Карточка не найдена.

И кнопку назад.

13. Обработка данных

Перед выводом:

services → список строк

контакты → фильтровать пустые

текст → без лишних пробелов

не показывать пустые блоки

14. Роль UI

CARD VIEW:

получает card_id

запрашивает объект

формирует текст

отправляет сообщение

строит кнопки

Не:

валидирует данные

не редактирует объект

не читает файлы напрямую

15. Структура файлов
directory/
 ├ ui/
 │   ├ handlers.py
 │   ├ callbacks.py
 ├ render/
 │   ├ card_render.py
16. Инварианты step 5
1. карточка открывается только через loader
2. UI не читает JSON напрямую
3. карточка не перегружена
4. есть действия (запрос / контакт)
5. есть возврат назад
6. пустые поля не показываются
17. Ready check

Шаг готов, если:

1. карточка открывается по id
2. данные отображаются корректно
3. кнопка "запрос" работает или честно сообщает
4. кнопка "связаться" работает
5. кнопка назад возвращает к списку
6. UI не падает при ошибках
18. Смысл этого шага

Вот здесь DIRECTORY становится бизнесом:

человек → карточка → действие

До этого — просто структура.
Здесь начинается ценность.
