id
category

description (кратко)

contacts:
    phone
    whatsapp

10. ПРАВИЛА DATA

карточки не хранятся в коде

только JSON / база

единый формат

никаких “временных” полей

11. ЧТО ЗАПРЕЩЕНО
❌ разный формат карточек
❌ отсутствие обязательных полей
❌ хранение в разных местах
❌ "не партнёрские" карточки
❌ ручные правки в UI
12. INVARIANTS
1. все карточки = партнёры
2. единый стандарт карточки
3. данные приходят из одного источника
4. UI не содержит данных
5. карточка = точка действия
13. READY CHECK

Модуль готов, если:

1. есть категории
2. есть JSON карточек
3. список отображается
4. карточка открывается
5. есть кнопки действий
6. структура единая
7. нет "левых" данных
