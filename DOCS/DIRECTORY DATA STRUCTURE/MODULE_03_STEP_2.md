3. Что означает каждое новое поле
description_full

Полное описание карточки для открытия внутри бота и позже для сайта.

phone

Телефон, если нужен.
Если пока не используете — можно оставить пустую строку.

location

Короткая понятная локация.

languages

Какие языки доступны у партнёра.

services

Список услуг / специализаций.

image

Путь или URL изображения.
Пока можно оставить локальный путь, если он уже принят в проекте.

partner_status

Статус карточки.
На текущем этапе можно использовать:

partner

В будущем можно расширять, но сейчас не плодим варианты.

partner_chat_id

Под будущую доставку заявок.
Если не привязан — null

4. Что НЕ добавляем даже сейчас

Не добавляем:

actions
buttons
callback
ui_text
menu_label
логику показа
HTML/Markdown
внутренние служебные поля роутеров

То есть карточка должна остаться чистой data-моделью.

5. Правила для значений
languages

Массив строк:

["RU", "EN"]
services

Массив строк, без сложных вложенных объектов:

[
  "Стрижки каре",
  "Окрашивание"
]
partner_chat_id

Пока:

null

Не писать пустую строку, если это именно не подключённый chat_id.
